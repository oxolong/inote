# iNote 笔记文件规范 v1.0

## 概述

iNote 是一种基于本地文件的笔记体系规范。每一个体系就是一个文件夹，文件夹名即体系名。体系内每个概念节点对应一个 Markdown 文件。所有节点之间的关系记录在一个统一的 `_inote.json` 元数据文件中。

iNote 支持两种体系类型：

| 类型                   | 关系                 | 连线样式   |
| ---------------------- | -------------------- | ---------- |
| `knowledge` 知识体系 | 仅「包含」关系       | 无箭头连线 |
| `scenario` 场景体系  | 「动作」关系（有向） | 有向箭头   |

---

## 一、文件结构

### 知识体系

```
体系名/
├── _inote.json          # 元数据文件（必须）
├── 概念A.md              # 节点笔记
├── 概念B.md
├── 概念C.md
└── ...                  # 更多节点
```

### 场景体系

```
体系名/
├── _inote.json          # 元数据文件（必须）
├── 起点节点.md            # 节点笔记
├── 中间节点.md
├── 终点节点.md
├── ...                  # 更多节点
└── actions/              # 动作目录（必须）
    ├── from(A)to(B)act(X).md
    ├── from(B)to(C)act(Y).md
    └── ...
```

### 命名规则

- **节点文件**：`节点名.md`，节点名即文件名（不含扩展名）
- **动作文件**：`from(源节点名)to(目标节点名)act(动作名).md`，存放在 `actions/` 子目录中
- **元数据文件**：固定为 `_inote.json`，放在体系文件夹根目录

---

## 二、`_inote.json` 格式

### 知识体系

```json
{
  "version": "1.0",
  "type": "knowledge",
  "name": "体系名称",
  "description": "体系的简短描述（可选）",
  "nodes": [
    { "id": "n1", "name": "概念名称", "file": "概念名称.md" }
  ],
  "contains": [
    { "parent": "n1", "child": "n2" }
  ]
}
```

| 字段                  | 类型   | 必须 | 说明                                          |
| --------------------- | ------ | ---- | --------------------------------------------- |
| `version`           | string | ✅   | 规范版本，当前为 `"1.0"`                    |
| `type`              | string | ✅   | `"knowledge"`                               |
| `name`              | string | ✅   | 体系名称，与文件夹名一致                      |
| `description`       | string | ❌   | 体系描述                                      |
| `nodes`             | array  | ✅   | 节点列表                                      |
| `nodes[].id`        | string | ✅   | 节点唯一标识，如 `"n1"`, `"n2"`           |
| `nodes[].name`      | string | ✅   | 节点名称，与 `.md` 文件名（不含扩展名）一致 |
| `nodes[].file`      | string | ✅   | 节点对应的 Markdown 文件名                    |
| `contains`          | array  | ✅   | 包含关系列表                                  |
| `contains[].parent` | string | ✅   | 父节点 id                                     |
| `contains[].child`  | string | ✅   | 子节点 id                                     |

**规则：**

- `contains` 表示「父节点包含子节点」，前端展示为无箭头连线
- 一个父节点可以包含多个子节点，一个子节点也可以被多个父节点包含
- 节点 id 必须唯一，在 `nodes` 中定义，在 `contains` 中引用

---

### 场景体系

```json
{
  "version": "1.0",
  "type": "scenario",
  "name": "体系名称",
  "description": "体系的简短描述（可选）",
  "nodes": [
    { "id": "n1", "name": "节点名称", "file": "节点名称.md" }
  ],
  "actions": [
    {
      "id": "a1",
      "from": "n1",
      "to": "n2",
      "name": "动作名称",
      "file": "actions/from(节点1)to(节点2)act(动作名称).md"
    }
  ]
}
```

| 字段               | 类型   | 必须 | 说明                                             |
| ------------------ | ------ | ---- | ------------------------------------------------ |
| `version`        | string | ✅   | 规范版本，当前为 `"1.0"`                       |
| `type`           | string | ✅   | `"scenario"`                                   |
| `name`           | string | ✅   | 体系名称，与文件夹名一致                         |
| `description`    | string | ❌   | 体系描述                                         |
| `nodes`          | array  | ✅   | 节点列表（同上）                                 |
| `actions`        | array  | ✅   | 动作列表                                         |
| `actions[].id`   | string | ✅   | 动作唯一标识，如 `"a1"`, `"a2"`              |
| `actions[].from` | string | ✅   | 源节点 id                                        |
| `actions[].to`   | string | ✅   | 目标节点 id                                      |
| `actions[].name` | string | ✅   | 动作名称                                         |
| `actions[].file` | string | ✅   | 动作对应的 Markdown 文件路径（相对于体系根目录） |

**规则：**

- `actions` 表示有向动作，前端展示为有向箭头（from → to）
- 两个节点之间可以存在多个双向动作（如 A→B 和 B→A）
- 场景的起点和终点都是普通节点，不做特殊标记
- 循环结构通过 `actions` 自然形成回路（如 n3 → n1 即可形成循环）
- 动作文件统一存放在 `actions/` 子目录中
- 动作文件命名格式必须为：`from(源节点名)to(目标节点名)act(动作名).md`

## 三、Markdown 文件内容

每个 `.md` 文件是标准的 Markdown 格式，内容为该节点或动作的笔记。

节点文件示例 `CNN.md`：

```markdown
# CNN（卷积神经网络）

## 概述
卷积神经网络是一种深度学习模型，广泛应用于计算机视觉任务。

## 核心思想
- 局部感受野
- 权重共享
- 池化降采样

## 典型应用
- 图像分类
- 目标检测
- 语义分割
```

动作文件示例 `actions/from(需求分析)to(架构设计)act(方案设计).md`：

```markdown
# 方案设计

## 输入
需求分析文档

## 输出
系统架构设计方案

## 步骤
1. 分析需求要点
2. 确定技术选型
3. 设计模块划分
4. 输出架构图
```

### 公式书写规范

在 `.md` 文件中使用标准 LaTeX 语法书写数学公式，前端通过 KaTeX 自动渲染。

**行内公式**：使用单个 `$` 包裹，公式与正文在同一行。

```markdown
线性回归的数学表达式为 $y = w_1x_1 + w_2x_2 + ... + w_nx_n + b$，其中 $w_i$ 为权重。
```

**块级公式**：使用双 `$$` 包裹，公式独占一行居中显示。

```markdown
均方误差损失函数：

$$L = \frac{1}{N}\sum_{i=1}^{N}(y_i - \hat{y}_i)^2$$
```

**约束：**

- 公式必须使用标准 LaTeX 数学语法（如 `\frac`, `\sum`, `\hat`, `\sigma`, `\cdot`, `\odot` 等）
- `_` 和 `^` 用于下标和上标，花括号 `{}` 包裹多字符的上下标
- 不要在公式内部使用 Markdown 格式（如 `**` 加粗、`*` 斜体），会被误解析
- 不要使用 `$` 符号表示非数学含义（如货币金额），如需表示美元请转义为 `\$` 或使用中文「美元」

## 四、AI 生成指引

让 AI 生成 iNote 体系时，使用以下提示词模板：

> 请按照 iNote 规范，为「[体系名称]」生成一个完整的[知识/场景]体系。
>
> 要求：
>
> 1. 生成 `_inote.json` 元数据文件
> 2. 为每个节点生成对应的 `.md` 笔记文件
> 3. （场景体系）为每个动作生成 `actions/` 目录下的动作 `.md` 文件
> 4. 节点 id 使用 `n1`, `n2`, ... 格式
> 5. 动作 id 使用 `a1`, `a2`, ... 格式
> 6. 节点 `.md` 文件命名使用节点名，动作 `.md` 文件命名使用 `from(源节点名)to(目标节点名)act(动作名).md` 格式
> 7. 所有 `.md` 文件内容需包含有意义的笔记内容，而非空壳
>
> 请直接输出文件结构和内容。
