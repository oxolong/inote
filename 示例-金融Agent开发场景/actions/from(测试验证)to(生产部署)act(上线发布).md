# 上线发布

## 输入
- 通过测试验证的代码版本
- 测试报告
- 部署架构文档

## 输出
- 生产环境运行的服务
- 运维手册
- 应急预案

## 执行步骤
1. **构建镜像**：编译并打包 Docker 镜像
2. **推送到镜像仓库**：上传至私有镜像仓库
3. **更新 K8s 配置**：调整资源配置、副本数、自动扩缩容策略
4. **灰度发布**：
   - 10% 流量（内部用户，观察 24h）
   - 30% 流量（观察 24h）
   - 全量发布
5. **配置监控告警**：确保 Prometheus + Grafana 正常采集
6. **验证上线**：执行冒烟测试，确认核心功能正常

## 上线 Checklist
- [ ] 所有测试用例通过
- [ ] 安全渗透测试完成
- [ ] 回滚方案就绪（保留上一版本的 Deployment）
- [ ] 数据库备份完成
- [ ] 值班人员确认

## 回滚策略
如果上线后 1 小时内错误率 > 5%，立即回滚至上一版本。
