# 单元测试

## 输入
- 各模块源代码
- 接口规范文档

## 输出
- 单元测试报告
- 代码覆盖率报告
- 模块 Bug 清单

## 执行步骤
1. **编写测试用例**：针对每个函数/方法设计测试
2. **Mock 外部依赖**：模拟 LLM API、数据库等外部服务
3. **执行测试**：运行 pytest 或类似框架
4. **检查覆盖率**：确保代码覆盖率 > 80%
5. **修复发现的问题**：逐一处理测试失败的用例

## 金融场景专项测试
- Prompt 注入攻击防护测试
- 金融计算精度验证（浮点数处理）
- 敏感信息泄露检查

## 通过标准
- 所有测试用例通过
- 覆盖率达到 80% 以上
- 无 P0/P1 级别 Bug
