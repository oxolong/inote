# 方案设计

## 输入
- 产品需求文档（PRD）
- 用户故事与功能清单
- 数据源清单

## 输出
- 系统架构设计文档
- 技术选型方案
- 模块划分与接口规范

## 执行步骤
1. **梳理功能边界**：确定 Agent 需要具备的核心能力
2. **选择架构模式**：Multi-Agent vs 单 Agent + 工具调用 vs RAG Pipeline
3. **技术选型**：确定 LLM 基座、向量数据库、缓存、消息队列等组件
4. **数据流设计**：绘制从用户输入到 Agent 输出的完整数据流
5. **安全设计**：定义 API 安全策略、数据脱敏方案、权限模型
6. **输出架构图**：使用 C4 模型或类似方法绘制架构图

## 注意事项
- 金融场景对数据安全和合规有极高要求，需在架构层面考虑
- 提前评估各方案的成本（Token 消耗、基础设施费用）
