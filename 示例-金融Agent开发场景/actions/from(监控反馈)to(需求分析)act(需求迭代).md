# 需求迭代

## 输入
- 用户反馈数据
- 系统监控指标
- 运营数据报告

## 输出
- 优化需求清单（按优先级排序）
- 产品迭代计划
- 改进方案建议

## 执行步骤
1. **收集反馈**：汇总用户反馈、监控数据、运营数据
2. **问题归类**：
   - 功能缺失：用户需要但系统不支持的能力
   - 准确性问题：回答错误率高的问题类型
   - 性能问题：响应慢、超时频繁的场景
   - 体验问题：交互流程不顺畅的环节
3. **优先级评估**：
   - P0（紧急）：影响核心功能的严重问题
   - P1（高）：显著影响用户体验的问题
   - P2（中）：改进空间明显的优化项
   - P3（低）：锦上添花的功能
4. **制定迭代计划**：确定下一个版本的迭代范围
5. **更新需求文档**：将确认的需求写入 PRD

## 反馈闭环
```
监控反馈 → 需求迭代 → 方案设计 → 编码实现 → 测试验证 → 上线发布 → 监控反馈
    ↑                                                                      ↓
    └──────────────────────── 持续循环 ───────────────────────────────────┘
```

## 迭代节奏建议
- 小版本（Bug 修复）：每周 1 次
- 中版本（功能优化）：每 2-4 周 1 次
- 大版本（新能力）：每季度 1 次
