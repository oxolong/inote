# 日志采集

## 输入
- 生产环境运行的 Agent 服务
- 监控基础设施（Prometheus、ELK 等）

## 输出
- 结构化日志数据
- 实时性能指标
- 用户行为分析数据

## 执行步骤
1. **配置日志采集**：
   - 应用日志 → Filebeat / Fluentd → Elasticsearch
   - 性能指标 → Prometheus → Grafana
2. **定义关键指标**：
   - 技术指标：QPS、延迟、错误率、Token 消耗
   - 业务指标：DAU、对话轮次、功能使用率
3. **搭建监控大盘**：Grafana Dashboard 可视化关键指标
4. **配置告警规则**：设定阈值，触发企业微信/钉钉/PagerDuty 通知
5. **日志分级存储**：
   - 热数据（近 7 天）：SSD 存储，快速查询
   - 温数据（7-30 天）：HDD 存储
   - 冷数据（> 30 天）：归档至对象存储

## 数据采集原则
- 脱敏处理：用户身份信息、账户信息在采集前脱敏
- 最小化采集：只采集必要数据，遵循数据最小化原则
