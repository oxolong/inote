# 缺陷修复

## 输入
- 测试报告中的 Bug 清单
- 用户反馈的问题列表

## 输出
- 修复后的代码
- 修复验证报告

## 执行步骤
1. **问题定位**：根据 Bug 描述和日志定位出问题的代码
2. **影响分析**：评估修复可能影响的其他模块
3. **编写修复代码**：在本地或开发分支进行修复
4. **回归测试**：运行完整测试套件，确保修复不会引入新问题
5. **代码审查**：修复代码需经过 Code Review 才能合入主分支
6. **更新测试用例**：为已修复的 Bug 添加对应的回归测试

## 常见缺陷类型
| 类型 | 示例 |
|------|------|
| 逻辑错误 | Prompt 模板中错误的推理步骤 |
| 边界条件 | 空输入、超长输入未处理 |
| 并发问题 | 多用户同时调用时的状态冲突 |
| 外部依赖 | 第三方 API 超时未处理 |

## 修复原则
- 治本不治标：修复根因而非表象
- 最小化改动：降低引入新问题的风险
- 添加测试：防止同类问题再次出现
